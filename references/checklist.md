# 代码审查检查清单

按语言/场景查阅，仅在需要时加载。

## 通用检查项

### 正确性
- [ ] 空值/nil/null 是否处理
- [ ] 数组越界、除零、整数溢出
- [ ] 循环的终止条件是否正确
- [ ] switch/match 是否有 default 分支
- [ ] 异步操作的错误回调是否处理
- [ ] 资源（文件句柄、连接、锁）是否正确释放
- [ ] 并发访问共享状态是否有竞态条件

### 安全性（OWASP Top 10 速查）
- [ ] SQL/NoSQL 注入：用户输入是否参数化
- [ ] XSS：输出到 HTML 是否转义
- [ ] 认证：敏感接口是否校验身份
- [ ] 授权：是否检查当前用户有权操作目标资源
- [ ] 敏感数据：密码/token/密钥是否明文存储或日志打印
- [ ] CSRF：状态变更请求是否有 token 校验
- [ ] 路径遍历：文件路径是否校验 `../`
- [ ] 反序列化：是否接受不可信来源的序列化数据
- [ ] 依赖：是否使用已知有 CVE 的库版本

### 性能
- [ ] 循环内是否有数据库查询（N+1）
- [ ] 大数据集是否使用分页/流式处理
- [ ] 是否有不必要的深拷贝
- [ ] 正则表达式是否可能 ReDoS
- [ ] 缓存是否有失效策略

### 可维护性
- [ ] 函数参数是否超过 5 个
- [ ] 嵌套层级是否超过 3 层
- [ ] 魔法数字是否提取为常量
- [ ] 是否有重复代码块（>10 行相同逻辑）

---

## Python 专项

- [ ] `except Exception` 是否过于宽泛
- [ ] 可变默认参数（`def f(x=[])`）
- [ ] `is` vs `==` 混用（仅 `None` 用 `is`）
- [ ] f-string 或 `.format()` 中是否有未转义的用户输入
- [ ] 上下文管理器（`with`）是否用于文件/锁/连接
- [ ] 类型提示与实际行为是否一致

## JavaScript/TypeScript 专项

- [ ] `===` vs `==` 混用
- [ ] Promise 是否有 `.catch()` 或 `try/catch`
- [ ] `async` 函数的返回值是否被 `await`
- [ ] 闭包中的循环变量捕获问题
- [ ] `innerHTML` 赋值是否经过 sanitize
- [ ] `eval()` / `new Function()` 是否使用了用户输入

## Java/Kotlin 专项

- [ ] `equals()` vs `==` 混用（对象比较）
- [ ] `NullPointerException` 风险点
- [ ] `synchronized` 范围是否过大或过小
- [ ] Stream 是否在需要时关闭
- [ ] 字符串拼接在循环中是否用 `StringBuilder`

## Go 专项

- [ ] `error` 返回值是否检查
- [ ] goroutine 是否有泄漏风险（无退出机制）
- [ ] `defer` 在循环中是否会累积
- [ ] slice append 是否理解底层数组共享
- [ ] context 是否正确传递和取消

## SQL 专项

- [ ] 查询是否使用索引（EXPLAIN 验证）
- [ ] `SELECT *` 是否可以缩减字段
- [ ] 事务隔离级别是否合适
- [ ] 大表 JOIN 是否有笛卡尔积风险
- [ ] DDL 变更是否兼容在线迁移
